## Bracket Structures by Tier

### T1 — NBA (30 teams, Bo7 throughout)

Standard NBA-style bracket: 2 conferences, 8 teams each, 4 rounds.

| Round         | Series Format | Round Name    | Series ID Pattern                |
|---------------|---------------|---------------|----------------------------------|
| Round 1       | Bo7           | `Round1`      | `t1-r1-{east/west}-{seed}v{seed}` |
| Round 2       | Bo7           | `Round2`      | `t1-r2-{east/west}-{1/2}`       |
| Conf Finals   | Bo7           | `ConfFinals`  | `t1-cf-{east/west}`             |
| Finals        | Bo7           | `Finals`      | `t1-finals`                      |

Seeding: Top 8 per conference by W-L record (pointDiff tiebreaker). Matchups: 1v8, 2v7, 3v6, 4v5.

Home court: Higher seed in games 1, 2, 5, 7 (standard 2-2-1-1-1).

### T2 — NARBL (86 teams across 11 divisions, Bo3 division / Bo5 national)

Two-stage tournament: division playoffs then national tournament.

**Stage 1 — Division Playoffs (Bo3):**

| Round         | Series Format | Round Name    | Series ID Pattern                    |
|---------------|---------------|---------------|--------------------------------------|
| Division Semi | Bo3           | `DivSemi`     | `t2-div-{division-id}-s1` (1v4), `t2-div-{division-id}-s2` (2v3) |
| Division Final| Bo3           | `DivFinal`    | `t2-div-{division-id}-final`        |

Top 4 teams per division qualify. Division ID is the lowercase hyphenated division name (e.g., `great-lakes`).

**Stage 2 — National Tournament (Bo5):**

Once all 11 division finals are complete, `_checkT2NationalTournamentSeeding()` collects winners + top runners-up to fill a 16-team bracket, seeded by regular season record.

| Round         | Series Format | Round Name    | Series ID Pattern            |
|---------------|---------------|---------------|------------------------------|
| Nat Round 1   | Bo5           | `NatRound1`   | `t2-nat-r1-{1..8}`          |
| Nat Quarters  | Bo5           | `NatQuarter`  | `t2-nat-qf-{1..4}`          |
| Nat Semis     | Bo5           | `NatSemi`     | `t2-nat-sf-{1/2}`           |
| 3rd Place     | Bo3           | `ThirdPlace`  | `t2-3rd-place`               |
| Finals        | Bo5           | `Finals`      | `t2-finals`                  |

3rd place game exists for promotion determination (top 3 finishers + best regular season record are promoted to T1).

### T3 — MBL (144 teams across 24 metro divisions, Bo3 early / Bo5 late)

Multi-stage tournament with byes for top metro winners.

**Stage 1 — Metro Finals (Bo3):**
Top 2 from each of 24 metro divisions play a Bo3 series.

| Round         | Series Format | Round Name    | Series ID Pattern            |
|---------------|---------------|---------------|------------------------------|
| Metro Final   | Bo3           | `MetroFinal`  | `t3-metro-{division-id}`     |

**Stage 2 — Regional (Bo3):**
Once all 24 metro finals are complete, `_checkT3TournamentSeeding()` sorts winners by record. Top 8 get byes to Sweet 16. Teams 9–24 play a regional round (8 series: 9v24, 10v23, etc.).

| Round         | Series Format | Round Name    | Series ID Pattern            |
|---------------|---------------|---------------|------------------------------|
| Regional      | Bo3           | `Regional`    | `t3-regional-{1..8}`         |

**Stage 3+ — National Bracket (Bo5):**
8 bye teams + 8 regional winners form a 16-team bracket.

| Round         | Series Format | Round Name    | Series ID Pattern            |
|---------------|---------------|---------------|------------------------------|
| Sweet 16      | Bo5           | `Sweet16`     | `t3-sweet16-{1..8}`          |
| Quarterfinals | Bo5           | `Quarter`     | `t3-qf-{1..4}`              |
| Semifinals    | Bo5           | `Semi`        | `t3-sf-{1/2}`               |
| 3rd Place     | Bo3           | `ThirdPlace`  | `t3-3rd-place`               |
| Finals        | Bo5           | `Finals`      | `t3-finals`                  |

3rd place game exists for promotion determination (champion, runner-up, 3rd place winner promoted to T2).


## Playoff Data Structures

### `gameState.playoffData`

Generated by `OffseasonController.advanceToNextSeason()`. Contains bracket seeds per tier.

```
{
  t1: { type: 't1', bestOf: 7, east: [8 teams], west: [8 teams], ... },
  t2: { divisionBrackets: [{ division, seed1..seed4, ... }], ... },
  t3: { metroMatchups: [{ division, seed1, seed2 }], ... },
  initialized: true,
  completed: false  // set true when all 3 tiers' finals are complete
}
```

### `gameState.playoffSchedule`

Generated by `PlayoffEngine.generatePlayoffSchedule()`. Calendar of all potential games across all tiers.

```
{
  games: [ { id, seriesId, tier, round, gameNumber, bestOf, date,
             homeTeamId, awayTeamId, homeTeam, awayTeam,
             higherSeedId, lowerSeedId, played, skipped, result, boxScore } ],
  bySeries: { [seriesId]: [game, game, ...] },
  dates: [sorted unique dates]
}
```

Games are pre-generated for the maximum possible series length (e.g., 7 games for a Bo7). `PlayoffEngine.isGameNecessary()` checks whether a game still matters based on series state. Unnecessary games get `skipped: true` when encountered during simulation.

### `gameState.postseasonResults`

Built by `PlayoffSimController._buildPostseasonResults()` when all playoffs complete.

```
{
  t1: { champion, runnerUp, completed },
  t2: { champion, runnerUp, thirdPlaceWinner, thirdPlaceLoser, completed },
  t3: { champion, runnerUp, thirdPlaceWinner, thirdPlaceLoser, completed },
  promoted: { toT1: [3 teams], toT2: [3 teams] },
  relegated: { fromT1: [3 teams], fromT2: [3 teams] }
}
```

Promotion rules: T2 champion + best regular season record team + fill from playoff finishers. T3 champion + runner-up + 3rd place winner. Relegation: bottom 3 by regular season record.

## Entry and Exit Flows

### Entry: Season End → PlayoffHub

1. `GMMode.finishSeason()` or sim completes → `GameSimController.showSeasonEnd()`
2. `showSeasonEnd()` determines user's status (playoffs, stay, relegate), shows `SeasonEndModal`
3. User clicks "Begin Playoffs" → `window._seasonEndAdvanceCallback(action)`
4. `OffseasonController.advanceToNextSeason(action)`:
   - Sets phase to `POSTSEASON`
   - Closes SeasonEndModal via `window._reactCloseSeasonEnd()`
   - Generates brackets for all 3 tiers via `PlayoffEngine.generateT*Bracket()`
   - Generates playoff schedule via `PlayoffEngine.generatePlayoffSchedule()`
   - Determines user playoff status and first series ID
   - Saves state
   - Opens PlayoffHub via `window._reactShowPlayoffHub({ ...fullPayload })`

**Critical constraint**: `_showPlayoffSeriesStatus()` must call `window._reactPlayoffHubRefresh()`, never any modal opener. This prevents the old modal chain from hijacking the flow.

### Exit: PlayoffHub → Offseason

1. All playoffs complete → `PlayoffSimController._checkPlayoffsComplete()` sets `playoffData.completed = true` and builds `postseasonResults`
2. PlayoffHub shows "View Results" button
3. User clicks "View Results" → `handleViewResults()` builds modal data, calls `window._reactShowPlayoffEnd(modalData)`
4. `PlayoffEndModal` shows champions/results with "Begin Offseason" button
5. User clicks "Begin Offseason" → `window._playoffEndContinueCallback()` → `window._reactClosePlayoffHub()` → `data.onComplete()` → `OffseasonController.continueAfterPostseason()`
6. Promo/rel executes, OffseasonHub opens

### Resume from Save

`OffseasonController.resumeOffseason()` POSTSEASON case:
- Passes full payload from `gameState` (including `playoffSchedule`, `userSeriesId`, `userInPlayoffs`)
- PlayoffHub re-renders from live `gameState` data
- PlayoffSimController initializes lazily on first sim call

## Cross-Controller Wiring

PlayoffSimController and GameSimController are cross-referenced:
- `playoffSimController._gameSimController` — set during lazy init in game-init.js
- `gameSimController._playoffSimController` — set during lazy init in game-init.js

The cross-reference is needed for:
1. `watchCalendarPlayoffGame()` on GSC calls `psc._simOneScheduledGame()` for other games on the same date
2. `_closeCalendarPlayoffWatch()` on GSC calls `psc._updateBracketsAfterGames()`, `psc._updateUserSeriesForNextRound()`, `psc._checkPlayoffsComplete()`
3. `simToChampionship()` on PSC has a fallback to `gsc.simAllChampionshipRounds()` when no playoff schedule exists (legacy save compatibility)

Both share the same `ctx` object (gameState, helpers, engines) — they read/write the same `gameState.playoffSchedule` and `gameState.playoffData`.


## Design Decisions Made

1. **Calendar-based schedule**: All potential games are pre-generated with dates. `isGameNecessary()` determines which games actually need to be played. This replaces the old per-round modal chain.

2. **Three-tier parallel simulation**: All tiers play simultaneously on the same calendar. `simPlayoffDay()` sims all games on a date regardless of tier. `simPlayoffRound()` sims until the user's tier's current round completes.

3. **Dynamic seeding for later rounds**: T2 national and T3 regional/sweet16 brackets are populated dynamically once earlier stages complete. Placeholder games are generated at schedule creation time with empty team slots, filled by `_checkT2NationalTournamentSeeding()` and `_checkT3TournamentSeeding()`.

4. **Home court pattern**: Bo7 = 2-2-1-1-1, Bo5 = 2-2-1, Bo3 = 1-1-1 (higher seed hosts games 1, 3).

5. **Watch game ownership**: Watch infrastructure (`_watchGame`, `_watchTimer`, `_startWatchTimer`) stays on GameSimController because it's shared with regular season. Calendar playoff watch setup lives in `watchCalendarPlayoffGame()` on GSC, while result recording delegates bracket updates to PlayoffSimController.

6. **PlayoffHub is stable**: The UI was rebuilt in prior sessions and is working across all three tiers. The Phase 2 extraction changed only the controller routing, not the UI. Any future visual tweaks are a separate effort.

7. **Hub-era code retained**: The old per-round modal chain (`initBracketForHub`, `simAllChampionshipRounds`, T2/T3 continuation methods) remains in GameSimController as a fallback and for legacy save compatibility. It is a candidate for Phase 3 removal once confirmed unnecessary.

8. **3rd place games for promotion**: T2 and T3 both have 3rd place games between semifinal losers. The 3rd place finisher factors into promotion decisions.

9. **Promotion/relegation calculated in postseasonResults**: `_buildPostseasonResults()` determines promoted and relegated teams using a combination of playoff finish and regular season record. This data feeds into `OffseasonController.continueAfterPostseason()`.

10. **Single `mapStats` utility**: The box score stat mapper is defined once as `PlayoffSimController.mapStats()` and used by all playoff methods that build box scores. Previously copy-pasted 5 times.
